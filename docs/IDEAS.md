- Data generation: new script variant of `generate-vocab.ts` for exercise packs
- No UI changes needed

## 📖 Reader Improvements

### R1. Tap-to-Look-Up Word Tooltips

Tap any Arabic word in the reader to see its meaning in a small popover. The key insight: **start with exact-match from the text's own vocabulary list** (once QW7 below is done), not a full dictionary lookup.

Phase 1: match against `item.vocabulary[]` if present → popover with meaning + root
Phase 2: search across all loaded vocab sets
Phase 3: full dictionary (offline, bundled)

**Why this matters:** This is the #1 feature in LingQ and the most common request in language learning apps. Even a partial implementation (only words in the pre-vocab list) is immediately useful.

---

### R2. Vocabulary Panel Before Reading

Before the reader shows the full text, display 5–8 key words from that text. User can skip it and jump straight to reading.

**Data:** New optional `vocabulary[]` field in item JSON, generated by the Gemini pipeline:

```json
"vocabulary": [
  { "arabic": "سَافَرَ", "translation": "putovati", "root": "س ف ر" }
]
```

**UI:** A small step before `TextCard` — or as a collapsible "Key words" panel above the text.

---

### R3. Personal Notes Per Text

A small textarea below each text where the user can write observations, grammar notes, vocabulary they noticed. Saved to localStorage keyed by item ID (`fushalab_notes`).

- No generation changes needed
- Simple textarea, auto-saves on blur
- Shown in a collapsed accordion: "My notes (2)"

---

### R4. Listening Mode in Reader

A toggle that hides the Arabic text and plays the full TTS audio first. User tries to understand from audio only, then taps "Reveal text". Good for advanced learners who want a listening challenge from reading content.

- `isListeningMode` state flag in reader
- Hides text, shows large audio play button
- "Otkrij tekst" button after audio ends

---

### R5. Reading Timer (opt-in)

Track time spent per text. Display as a small clock icon in the reader. After marking complete, store `readingTimeMs` alongside the rating. Show average reading speed on the progress page ("prosječno 4 min/tekst").

## 🏗️ Infrastructure & Platform

### I1. PWA — Offline Support

The biggest quality-of-life improvement for mobile learners. Since all content is static JSON, a service worker can cache an entire level for offline use.

**Vercel serves static perfectly — PWA works with zero backend:**

1. Add `vite-plugin-pwa` (1 dependency)
2. Define a manifest: app name, icons, theme color, display: standalone
3. Cache strategy: cache-first for `/data/**` JSON, network-first for HTML
4. "Install app" prompt on mobile

**Why now:** The app is feature-complete enough that offline access adds real value. A learner on the metro can study without internet.

---

### I2. Export / Import Progress

Since there's no backend, progress lives only in the user's localStorage. If they switch browsers or devices, they lose everything. Solution: a simple "Export" button on the progress page that downloads `fushalab_backup.json` (all four localStorage keys), and an "Import" button that reads the file back.

- No backend needed
- `JSON.stringify` + `Blob` + `URL.createObjectURL` for export
- `FileReader` for import
- 30-second implementation for high trust value

---

### I3. Vercel Analytics

Vercel's free hobby tier includes **Web Analytics** (100K events/month) and **Speed Insights**. Adding it takes 2 lines:

```tsx
// main.tsx
import { Analytics } from '@vercel/analytics/react'
import { SpeedInsights } from '@vercel/speed-insights/react'
```

**What you'd learn:**
- Which reading categories are most used
- Which exercise packs get started vs abandoned
- Bounce rate from home → reading
- Mobile vs desktop split
- Real-world performance on different devices

This is genuinely useful for deciding what to build next — instead of guessing.

---

### I4. Vercel KV — Cross-Device Progress Sync

Vercel KV (Redis) is available on the free hobby tier: **30MB storage, 30K daily reads**.

With a single Edge Function + KV, you can offer optional anonymous sync:
- On first visit: generate a random `userId` (UUID), store in localStorage
- "Sync to cloud" button: sends all progress to `kv.set(userId, progress)`
- On new device: enter your sync code → restores progress
- No accounts, no passwords, no email — just a UUID

30MB is more than enough for thousands of users' localStorage data (each user's progress is <50KB).

**This is the one Vercel feature that unlocks real long-term value for learners who switch devices.**

---

### I5. Vercel Edge Functions — Content Search API

A lightweight API route (`/api/search?q=...`) that searches across all reading text titles and summaries. Since Vercel Edge Functions run on every request with no cold start, it's fast enough for a search-as-you-type experience.

Alternative (simpler): bundle a search index at build time using `flexsearch` — zero API calls, instant offline search.

## 🎓 Learning Experience

### L1. Onboarding — Level Placement Quiz

A short 3-question quiz on first visit. Show one sentence from B1, B2, C1. Ask "Do you understand this?" Map answers to a recommended starting level. Store in localStorage, redirect to that level. Skippable.

- No data changes needed (use existing text sentences)
- Big UX value for new users who don't know where to start

---

### L2. Grammar Reference Section

A lightweight `/grammar` page — not interactive, just a clean reference. Tables for:
- Verb conjugation patterns (perfect/imperfect)
- Dual forms
- Broken plurals (most common patterns)
- إضافة construction
- Relative clauses (الَّذِي / الَّتِي)

Each table has an example sentence that links to a reading text in the app where that pattern appears. Makes the app useful as a grammar companion, not just content browser.

**Why not an external resource:** Having it in-app keeps the learner in context. They can jump from a confusing sentence in the reader to the grammar reference and back.

---

### L3. Smart Home Page — Return User Experience

The home page currently shows the same content to everyone. For returning users (detected by any progress in localStorage), replace the hero with a personalized dashboard:

```
مَرْحَبًا بِعَوْدَتِكَ — Welcome back

[🔥 5-day streak]  [📖 Continue: Travel B2 · text 14]  [⚡ 340 XP]

Today's goal: ●●○○○  (2 of 5 texts)
Word of the Day: ↓ (below)
```

The current word-of-day + sections grid stays below. But the top becomes about *your* journey, not a generic intro.

---

### L4. "Similar Texts" on Reader Completion

After finishing a text, show 2–3 suggestions from the same category or level:

> _"Pročitaj sljedeće iz Travel B2:"_
> _→ "Kroz pustinjska sela..." | "Na aerodromu..." | "Hotel pored mora..."_

Derived entirely from the existing index. No new data needed. Makes it easy to keep going instead of navigating back manually.

---

### L5. Vocabulary from Reading — Save Words While Reading

In the reader, when a word tooltip appears (R1), add a "💾 Sačuvaj" button. Saved words go to a personal word list in `fushalab_saved_words`. Accessible from the vocabulary page as a new tab "Saved Words" alongside the regular sets.

The saved list becomes a personalised vocabulary set the learner built themselves — much more meaningful than pre-packaged sets.

## 📦 Content Pipeline

### C1. Vocabulary Extraction in Generation

Extend the Gemini generation prompt to also extract 5–8 key vocabulary items from each generated text. Add to item JSON:

```json
"vocabulary": [
  { "arabic": "اِسْتَكْشَفَ", "translation": "istraživati", "root": "ك ش ف" }
]
```

This feeds R1 (word tooltips) and R2 (pre-reading panel) with zero extra runtime cost.

---

### C2. Expand "Work" Category

`public/data/work/` has only 11 files — a stub. Business, professional, and workplace Arabic is genuinely useful for learners. Generate 40–50 texts at B2/C1 covering: meetings, negotiations, CV writing, business correspondence, office situations.

---

### C3. More Exercise Packs — Reading-Based (B1–C2)

Current exercise packs are A1–B2 vocabulary-topic packs. Add reading-comprehension exercise packs tied to actual reading categories:

- Travel B1 Pack: 10 questions about travel texts
- News B2 Pack: factual comprehension from news texts
- Literature C1 Pack: inference and meaning questions

New exercise type to add: **"True/False" (صَحِّح أَوْ خَطَّأ)** — a statement about a text, user taps True or False with an explanation.

---

### C4. Proverbs Category (أمثال)

Short Arabic proverbs with cultural explanation — perfect for B1/B2 short-form reading. Each entry: the proverb (1 line), its literal meaning, its idiomatic meaning, and a usage example. Very shareable, very memorable.

Example: _مَنْ جَدَّ وَجَدَ_ — "He who strives, finds." = Ustraj i uspijet ćeš.

---

### C5. More Conversation Dialogues

Current dialogues cover formal/B1-B2 situations. Add A1 beginner dialogues for:
- Greetings and introductions (full set)
- Numbers, shopping, prices
- Asking for directions
- Family and personal info
- Weather and small talk

These connect the conversation section to the exercise vocabulary content (same topics, different format).

## 🔭 Big Features (v2 thinking)

These are significant enough to plan separately — not quick wins.

### B1. User Accounts + Sync (Full)

If the app grows beyond personal use, Vercel KV (I4) scales to a proper account system with email + password via a lightweight Edge Function. Progress, saved words, notes — all synced. Still no heavy backend, all Edge Functions + KV.

Not needed now — I4's anonymous UUID sync solves 90% of the use case.

---

### B2. AI Reading Assistant

An AI-powered "explain this sentence" button in the reader. User taps a sentence → sends it to an API (could be user's own API key, like the generation pipeline) → gets a grammatical breakdown and explanation in Bosnian.

**Constraint:** Requires an API key, either bundled (paid) or user-supplied. The generation pipeline pattern (user provides Gemini key) is the right model.

---

### B3. Spaced Repetition for Reading

A proper SRS algorithm (like SM-2) applied to reading texts. Instead of just surfacing "hard" texts after 3 days (QW5), calculate optimal review intervals per item. Would require its own localStorage key with interval/ease/due-date data per item.

This makes the app more like a reading-focused Anki. High value, medium complexity.

---

### B4. Certificate of Completion

When a learner finishes all texts in a level (e.g. all B1 Travel — 100 texts rated ≥ easy), generate a simple in-browser SVG certificate with their name (entered on first use or at that moment), the level, and the date. Downloadable as PNG via `html2canvas`.

Small effort, big emotional reward.

---

### B5. Text-as-Image Export

Generate a shareable card from any reading text — Arabic text + translation + app branding — for sharing to Instagram/WhatsApp. Canvas API or `html2canvas`. Cultural Arabic calligraphy aesthetic.
